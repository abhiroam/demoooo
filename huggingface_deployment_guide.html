🤗 Hugging Face Spaces Deployment Guide

This guide will walk you through deploying your CausalTraj EEG application to a Hugging Face Space. Hugging Face Spaces is a 100% free hosting platform specifically designed for Machine Learning models, and provides an incredible 16GB of RAM and 2 vCPUs completely free, with no strict HTTP timeouts.

Important Prerequisite: The required Dockerfile has been generated in your project folder. Let's push this directly to a Hugging Face Space.
1 Create a Hugging Face Account

Go to huggingface.co/join and create a free account if you don't already have one.

2 Create a New Space
  1. Go to huggingface.co/spaces and click the "Create new Space" button at the top right.
  2. Give your Space a name (e.g., causaltraj-eeg).
  3. Under "Select the Space SDK", you MUST choose "Docker".
  4. Under "Docker template", select "Blank".
  5. Keep the "Space Hardware" on "Free" (2 vCPU, 16 GB RAM).
  6. Click "Create Space".
3 Upload Your Files

Your new Space is essentially a Git repository hosted by Hugging Face. You can push your code to it! There are two ways to get your code into the space:

Option A: Upload via the Browser (Easiest if not using Git CLI)

  1. In your new Space, click on the "Files" tab near the top.
  2. Click the "Add file" dropdown button and select "Upload files".
  3. Drag and drop ALL the files and folders from your demoooo folder (including Dockerfile, api.py, the frontend folder, and your model_best.pth checkpoint) into the upload area.
  4. Scroll down, optionally add a commit message, and click "Commit changes".

Option B: Push via Git

You can add the Hugging Face Space as a remote to your existing Git repository and push directly to it:

git remote add space https://huggingface.co/spaces/YOUR_USERNAME/YOUR_SPACE_NAME
git push space main

(You will need an Access Token from your Hugging Face settings to use as your password when pushing via Git).

4 Watch it Build!

As soon as you commit or push your code, Hugging Face will automatically detect the Dockerfile and begin building your container.

  • Click the "App" tab to see the live console logs.
  • It will download PyTorch and install your requirements.
  • Once "Building" changes to "Running", your dashboard will appear directly on the page!
5 Verifying the App

In the logs, look for the following line to ensure your trained models loaded properly:

✅ Loaded trained weights from .../model_best.pth

If you see ⚠️ No checkpoint found, it means you forgot to upload your model_best.pth file to the Space!

Need to share it?

Your app is now live! Simply share the URL link of your Space with anyone. They can use the "Live EEG Upload" tab, and they won't experience the 30-second crash you had on Render since Hugging Face allows for heavy ML processing.
